    Modelos de Sofware

    Para desarrollar cualquier tipo de programa o software, se deve aplicar un modelo de sistema de vida del sofware con el fin de facilitar una metologia entre estas:


    • Modelo en Cascada
    • Modelo Repetitivo
    • Modelo en Espiral
    • Modelo en 'V'
    • Modelo Big Bang

    Modelo en cascada:

    Este toma las actividades fundamentalles del proceso de operacion, validacion y ls complementa con fases del proceso como la especificacion de requerimiento, diseño de software, implementacion y pruebas.

    Modelo Repetitivo:

    Este modelo permite al usuario de desarrollo de softare ediante repeticiones haciendo referencia del modelo de desarrolo siquico para repetir el ciclo de vida del softare.

    Modelo en Espiral:

    Este modelo espiral es una combinacion del modelo en cascada y el odelo Repetitivo, dnde se tiene el riesgo de estar, si epieza fijando los objetivos y las limitaciones al epezar cada repeticion en la etapa siguiente cosnta en crear los productos.

    Modelo en 'V':

    Modelo en V uno de los grandes probleas del modelo casado es que solo se basa en las siguientes fases y no se puede volver atras si hay errores.

    Modelo Big Bang:

    Es uno de los modelos mas simples ya que nesesita poca planificacion, mucha programacion y muchos fondos. Este modelo tiene como concepto principal la creacion del universo considerando el mejor producto de softare.
